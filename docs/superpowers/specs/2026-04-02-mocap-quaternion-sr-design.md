# Design: Quaternion Symbolic Regression on CMU Motion Capture Data

**Date:** 2026-04-02
**Status:** Approved

## Goal

Validate the multi-typed `PTree` symbolic regression framework on a real problem
where type safety provides a genuine combinatorial advantage: recovering the
quaternion integration formula `exp_q(ω·dt/2)` from motion capture data.

This is a controlled experiment — the ground truth is known, so success and
failure are unambiguous.

## Problem Statement

Given angular velocity `ω :: Vec3` and timestep `dt :: Double`, find a
`PTree MocapInput Quaternion` that approximates the delta quaternion `Δq`
between consecutive frames of a CMU motion capture clip.

Ground truth: `Δq = exp_q(scale_v(0.5 * dt, ω))`

The type system prevents the search from constructing meaningless expressions
(e.g. adding a quaternion to a scalar), reducing the effective search space.

## Components

### 1. Python Preprocessing (`scripts/preprocess_mocap.py`)

- **Input:** single BVH file from CMU mocap database, root hip joint
- **Steps:**
  1. Parse BVH (bvhio or bvh-tools)
  2. Extract Euler angles per frame, convert to quaternion sequence
  3. Enforce quaternion continuity: flip sign if `q[i] · q[i-1] < 0`
  4. Compute delta quaternion: `Δq[i] = conjugate(q[i-1]) * q[i]`
  5. Compute angular velocity: `ω[i] = 2 * log_q(Δq[i]) / dt`
  6. Validate: reconstruct `q[i]` from `ω[i]` via `exp_q(ω·dt/2)`, assert mean
     quaternion distance < 1e-6
- **Output:** `data/mocap_sample.csv` (~3000–5000 rows)
- **CSV columns:** `omega_x, omega_y, omega_z, dt, dq_w, dq_x, dq_y, dq_z`

### 2. Types (`main-mocap.hs`)

```haskell
type Vec3 = V3 Double
type Quat = Quaternion Double

data MocapInput = MocapInput { omega :: Vec3, dt :: Double }
```

### 3. Minimal Function Pool

| Name     | Type                  | Implementation                                      |
|----------|-----------------------|-----------------------------------------------------|
| `omega`  | leaf → `Vec3`         | `omega` field of `MocapInput`                       |
| `dt`     | leaf → `Double`       | `dt` field of `MocapInput`                          |
| `exp_q`  | `Vec3 → Quat`         | `\v -> let h = norm v in Quaternion (cos h) (sinc h *^ v)` |
| `scale_v`| `Double → Vec3 → Vec3`| `(*^)` from `linear`                                |
| `add_v`  | `Vec3 → Vec3 → Vec3`  | `(+)`                                               |
| `q_mul`  | `Quat → Quat → Quat`  | `(*)`                                               |
| `neg_v`  | `Vec3 → Vec3`         | `negate`                                            |
| `norm_v` | `Vec3 → Double`       | `norm`                                              |
| `PConst` | `Double`              | existing mutable `IORef Double` (coordinate descent)|

The ground truth path through the tree:
```
exp_q( scale_v( (*) (PConst≈0.5) dt,  omega ) )
```

### 4. Energy Function

```haskell
energyMocap :: V.Vector (Vec3, Double, Quat) -> PTree MocapInput Quat -> IO Double
energyMocap pts tree = do
    errs <- forM (V.toList pts) $ \(w, d, q_target) -> do
        let inp = MocapInput w d
        q_hat <- evalTree tree inp
        let d_val = q_hat `dot` q_target
        return (1 - d_val*d_val)
    return $ sum errs / fromIntegral (length errs)
```

Dot-product loss `1 - |q̂ · q|²` handles the quaternion double-cover and is
smooth everywhere. Minimum is 0 (perfect match), maximum is 1.

### 5. Search Configuration

```haskell
data SearchConfig = SearchConfig
  { maxWallSeconds  :: Maybe Int      -- Nothing = unlimited
  , targetEnergy    :: Maybe Double   -- stop when energy < threshold
  , maxRounds       :: Int            -- hard iteration cap
  , stepsPerSwap    :: Int
  , logEvery        :: Int            -- print status every N rounds
  , checkpointFile  :: Maybe FilePath -- save best tree + energy here
  , checkpointEvery :: Int            -- rounds between checkpoints
  }
```

### 6. Parallelism

The parallel tempering chains are independent between swap phases. Each chain's
`stepsPerSwap` MC steps run concurrently via `mapConcurrently` (from `async`),
then execution synchronises for the swap step. No STM required.

### 7. Progress Output

Printed every `logEvery` rounds:
```
[round 142/2000 | 00:04:32 elapsed] best=0.00341  chain temps: 0.001 0.003 0.01 ...
```

### 8. Checkpointing

Appended to `checkpointFile` every `checkpointEvery` rounds:
```
round,energy,tree
142,0.00341,"exp_q(scale_v((*) 0.4821 dt) omega)"
```

Allows post-hoc analysis of the energy walk and inspection of intermediate
expressions.

### 9. Stopping Conditions

The search loop exits on whichever triggers first:
- `maxRounds` reached
- Wall time exceeds `maxWallSeconds`
- Best energy drops below `targetEnergy`

The final output reports which condition stopped the run.

## New Dependencies

```
linear    -- Vec3, Quaternion types and operations
async     -- mapConcurrently for parallel chains
time      -- wall-clock stopping condition
```

## Success Criteria

The experiment succeeds if the discovered tree is semantically equivalent to
`exp_q(scale_v(c * dt, omega))` with `c ≈ 0.5` and energy < 0.001.
Partial success (correct structure, wrong constant) is still informative.
